1. Introduction

This report presents the design and implementation of a Library Management System developed as part of the Software Engineering Internship Assignment. The objective of this project was to build a full-stack web application that allows users to manage books through CRUD (Create, Read, Update, Delete) operations.

The backend was developed using ASP.NET Core 9 Web API with SQLite and Entity Framework Core, while the frontend was implemented using React with TypeScript.

2. Project Overview

  • Create new book records
  • View all existing books
  • View a specific book by ISBN
  • Update book details
  • Delete book records

3. System Architecture

The application follows a client-server architecture:

  • Frontend: React + TypeScript
  • Backend: ASP.NET Core 9 Web API
  • Database: SQLite

4. Backend Implementation

Technologies Used

  • ASP.NET Core 9 Web API
  • SQLite
  • Entity Framework Core
  • AutoMapper
  • Swagger / OpenAPI

API Endpoints

Method Endpoint Description
GET/api/BooksRetrieve all books
GET/api/Books/{isbn}Retrieve book by ISBN
POST/api/Books/create-bookCreate new book
PUT/api/Books/{isbn}Update book
DELETE/api/Books/{isbn}Delete book

Key Backend Features

  • Service Layer architecture
  • DTO pattern implementation
  • AutoMapper for object mapping
  • CORS configuration
  • Swagger documentation

6. Challenges Faced

  • CORS Policy Issue
    Frontend and backend ran on different ports, causing cross-origin issues. This was resolved by configuring CORS middleware in the backend.
  • API Integration Issues
    Ensured correct usage of GET by ISBN and PUT endpoints by verifying functionality using Swagger.
  • UI Improvements
    Replaced direct delete button with an “Options” page for better user experience.
  • Type Safety
    Improved TypeScript typing and removed unused variables reported by ESLint.
